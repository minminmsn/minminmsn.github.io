追寻知识源头

Posted on 2018-11-12 Edited on 2023-05-26 In mind

一手信息

致知在格物，格物而后知至

二手信息

著名解说有二 一是朱熹版如下： 所谓致知在格物者，言欲致吾之知，在即物而穷其理也。盖人心之灵莫不有知，而天下之物莫不有理，惟于理有未穷，故其知有不尽也。是以大学始教，必使学者即凡天下之物，莫不因其已知之理而益穷之，以求至乎其极。至于用力之久，而一旦豁然贯通焉，则众物之表里精粗无不到，而吾心之全体大用无不明矣。此谓物格，此谓知之至也。 二是王阳明版如下： 物者，事也，凡意之所发必有其事，意所在之事谓之物。格者，正也，正其不正以归于正之谓也。归於正者，为善之谓也。夫是之谓格。心者身之主，意者心之发，知者意之体，物者意之用。如意用於事亲，即事亲之事，格之必尽。夫天理则吾事亲之良知，无私欲之间，而得以致其极。知致则意无所欺，而可诚矣；意诚则心无所放，而可正矣。格物如格君之格，是正其不正以归於正。格物是止至善之功，既知至善，即知格物矣。

三手信息

对解说的解说 牛顿：格苹果落地而致万有引力之知； 徐爱：心犹镜也。圣人心如明镜，常人心如昏镜。近世格物之说，如以镜照物，照上用功，不知镜尚昏在，何能照？先生之格物，如磨镜而使之明，磨上用功，明了后亦未尝废照； ……

追寻知识源头

一手信息太难懂，二手信息也难对付，只好迎合三手信息，但三手信息就像道生一，二生三，三懂得再多也只是局部片面，而且此生有限，用有限的时间去追逐万万个局部显然不智，也不会达到头部，最终终日忙忙碌碌而无所作为！

Posted on 2018-11-09 Edited on 2023-05-26 In bigdata

参考

https://www.elastic.co/guide/en/elasticsearch/reference/5.0/modules-snapshots.html https://www.elastic.co/guide/en/elasticsearch/guide/current/\_rolling\_restarts.html https://blog.csdn.net/u014431852/article/details/52905821

环境

阿里云elasticsearch集群5.0版本 微软云elasticsearch集群5.6版本

需求

需要把阿里云elasticsearch集群新老数据迁移到微软云elasticsearch集群

解决

新数据比较好弄数据源输出到新的微软云kafka集群然后微软云logstash消费新数据到新elasticsearch集群即可，关于老数据迁移比较麻烦，但官网也给了成熟的解决方案既是快照备份与还原，下面实施过程既是对实施过程的记录

实施

阿里云elasticsearch集群操作

一，先关闭数据平衡，注意一个一个的来，关一个节点的进程none,all循环一次，否则最后集群切片变动，恢复时间很长

1、修改elasticsearch.yml配置，添加如下 path.repo: /storage/esdata 设置索引备份快照路径 注意所有的master节点与data节点都需要配置

2、关闭自动平衡

curl -XPUT http://10.10.88.86:9200/\_cluster/settings -d’ { “transient” : { “cluster.routing.allocation.enable” : “none” } }’ elasticsearch日志 [root@elk-es01 storage]# tail -f es-cluster.log [2018-05-11T15:24:15,605][INFO ][o.e.c.s.ClusterSettings ] [elk-es01] updating [cluster.routing.allocation.enable] from [ALL] to [none]

3、重启elasticseach

4、启动自动平衡

curl -XPUT http:///10.10.88.86:9200/_cluster/settings -d’ { “transient” : { “cluster.routing.allocation.enable” : “all” } }’

5、重复循环1、2、3、4操作在剩下几个节点，最好通过/_cat/health查看集群恢复100%后操作下一次，如果不关闭allocation重启elasticsearch集群恢复时间很长会从0%开始，关闭后从节点数减一除以总节点数比例开始恢复会块很多

二、设置快照基本配置

四、屏蔽IP

  • 回到正见，我想这个问题是个开放性的问题，每个人都会有不同的理解。正见大家（包括我）可能还不太清楚，那偏见这个词总还是有点印象的吧？何为偏见，关键在这一个偏字，他或者她以及它可能看到或者听到再或者知道某些你不知道的事物，所以会心理上感觉这个事哈你了解的没有我的全面，或者我这里有你所不知道的，显然你在见识就不全面，所以就产生了偏见。但是谁又能保证自己所知道的不是其中一部分呢，还有些东西我们不知道我们不知道呢，所以说偏见是永恒的。

如何提高自己的创造力

Posted on 2018-11-09 Edited on 2023-05-26 In mind , talk

读了肯·威尔伯的生活就像练习，里面有很多方法技巧可以直接拿来使用，是授人以渔的。现在书籍这么多，授人以渔的还是挺少的，许多书籍要么是展示性的，要么很有局限性不适合大众推广。今天我在这里抽取书中一个小片段来分享，很适合大家来提升自己的创造力！

练习下面四个步骤来提升你的创造力：

  • 一，吸收：首先聚焦于一个你喜欢或者关注的点或领域，然后全身心投入进去！你可以通过读书、拜访专家、或者直接参加活动来吸收知识，至少得坚持做一种。在你想站在巨人的肩膀上前，先跟他们做朋友！

  • 二，酝酿：现在放下你已经吸收的东西，改变环境。比如你加班很晚，苦苦思索一些问题的解决方案终无所获，此时你应该赶紧回家！再比如你周末在家看了一整天的书，晚上你可以去看场电影！最后比如你在一个地方呆腻了，那就独自去旅行吧！这些都是改变环境，暂时的放下，然后继续向前，接触不同的状态和身体，从而了解不同的观点。这迟早会产生火花，点燃创造之屋。这是个量变的过程，就好像荷花定理一样，荷塘里面的荷花每天成倍增长，在开满前一天的量只有最后一天的一半，所以这个过程需要坚持下来，在没有达到圆满前都是很少的！

  • 三，灵感：经过前两个步骤的练习，现在创造性的观点随时可能不期而至，特别是3B的地方（bathroom、bed、bus）。另外深夜和清早往往是人们特别有灵感的时刻，因为这个时间段人们介于清醒与睡眠之间，此时梦境里面的天马行空会与现实清醒的肉身状态交汇最容易产生创造性的想法，当然这些想法与前面的吸收和酝酿是相关的。想法来临时要立即丝毫不漏的记录下来。我们有时会有这样的经历，心里想了一段非常精彩的对话，过了那个时间段你可能忘了个精光或者再也没有兴致去回想了，所以此时要迅速地记录下来才不至于与灵感插肩而过！

  • 四，评估：此时，你已经产生了一些想法了，可以从多角度来分析你的领悟，知道它如何产生作用，继续批判性地评价你的观点，看看能不能再完善它？能不能成功的实现它？

以上就是通过日常整合式练习来提升自己的创造力，通过练习势必会提高自己的创造力。创造性的观点、方法、技术等就是这样来的，他们来源于旧的事物，又诞生了新的东西，实际上这就是进化的本质，值得大家好好的去练习！

知行合一的五种含义

Posted on 2018-11-09 Edited on 2023-05-26 In mind

知行合一四字至少包涵五层含义：

一是知便是行

一念发动处，便即是行了。譬如发动处有不善，就将这不善的念克倒了，须要彻根彻底，不使那一念不善潜伏在胸中。

二是不知不

譬如闻恶臭属知，恶恶臭属行。只闻那恶臭时已自恶了，不是闻了后又立个心去恶。如鼻塞人虽见恶臭在前，鼻中不曾闻得，便亦不甚恶，亦只是不曾知臭。

三是不行不知

未有知而不行者；知而不行，知是未知。譬如味之美恶，必待入口而后知，岂有不待入口而已先知食味之美恶者邪？ 又如路途之险夷，必待身亲履历而后知，岂有不待身亲履历而已先知路途之险夷者耶？

四是知易行艰

譬如称某人知孝，某人知弟，必是其人已会行孝行弟，方可称他知孝知弟。不成只是晓得说些孝弟的话，便可称为知孝知弟。

五是知行合一

凡谓之行者，只是着实去做这件事。譬如若着实做学问思辩的工夫，则学问思辩亦便是行矣。学是学做这件事，问是问做这件事，思辩是思辩做这件事，则行亦便是学问思辩矣。若谓学问思辩之，然后去行，却如何悬空先去学问思辩得？行时又如何去得做学问思辩的事？行之明觉精察处，便是知；知之真切笃实处，便是行。若行而不能精察明觉，便是冥行，便是“学而不思则罔”，所以必须说个知；知而不能真切笃实，便是妄想，便是“思而不学则殆”，所以必须说个行；元来只是一个工夫。

名称：Hadoop 官网：http://hadoop.apache.org/ 简介：Hadoop是一个由Apache基金会所开发的分布式系统基础架构。用户可以在不了解分布式底层细节的情况下，开发分布式程序。充分利用集群的威力进行高速运算和存储。 Hadoop实现了一个分布式文件系统（Hadoop Distributed File System），简称HDFS。HDFS有高容错性的特点，并且设计用来部署在低廉的（low-cost）硬件上；而且它提供高吞吐量（high throughput）来访问应用程序的数据，适合那些有着超大数据集（large data set）的应用程序。HDFS放宽了（relax）POSIX的要求，可以以流的形式访问（streaming access）文件系统中的数据。Hadoop的框架最核心的设计就是：HDFS和MapReduce。HDFS为海量的数据提供了存储，则MapReduce为海量的数据提供了计算。

名称：Python 官网：https://www.python.org/ 简介：Python 是一种面向对象的解释型计算机程序设计语言,是纯粹的自由软件， 源代码和解释器CPython遵循 GPL(GNU General Public License)协议。Python语法简洁清晰，特色之一是强制用空白符(white space)作为语句缩进。Python具有丰富和强大的库。它常被昵称为胶水语言，能够把用其他语言制作的各种模块（尤其是C/C++）很轻松地联结在一起。常见的一种应用情形是，使用Python快速生成程序的原型（有时甚至是程序的最终界面），然后对其中有特别要求的部分，用更合适的语言改写，比如3D游戏中的图形渲染模块，性能要求特别高，就可以用C/C++重写，而后封装为Python可以调用的扩展类库。需要注意的是在您使用扩展类库时可能需要考虑平台问题，某些可能不提供跨平台的实现。

13、自动化运维

16、持续集成

19、私有镜像库

名称：Harbor 官网：https://github.com/vmware/harbor 简介：Harbor是一个用于存储和分发Docker镜像的企业级Registry服务器，通过添加一些企业必需的功能特性，例如安全、标识和管理等，扩展了开源Docker Distribution。作为一个企业级私有Registry服务器，Harbor提供了更好的性能和安全。提升用户使用Registry构建和运行环境传输镜像的效率。Harbor支持安装在多个Registry节点的镜像资源复制，镜像全部保存在私有Registry中， 确保数据和知识产权在公司内部网络中管控。另外，Harbor也提供了高级的安全特性，诸如用户管理，访问控制和活动审计等。

跃迁：成为高手的技术

Posted on 2018-11-09 Edited on 2023-05-26 In note

古典

高手的三个境界

  • 见自己
  • 见世界
  • 见众生

高手的暗箱：利用规律，放大努力

  • 调用知识而非记忆知识
  • 从独自思考到联机的独立思考
  • 利用底层规律，通过法则，实现跃迁
  • 掌握自己注意力
  • 过滤信息，找到重点
  • 理解系统，能与陌生人联机协助
  • 大脑是用来观察、思考、创造和影响他人，不该用来记忆
  • 外包大脑，把不重要的技能外包，聚焦核心技能的跃迁

高手战略：在高价值区，做正确的事

  • 睡着以后，还能够持续迭代，需要打磨深思熟虑后做选择的战略能力，找到那些更少但是更好的事
  • 只打甜蜜区里的球、只求51%的胜率，需要定力与专注
  • 幂律分布（二八法则、长尾理论）：发现身边的高价值区，培优不补差
  • 头部效应：站位比努力更重要，从价值出发而非优势出发，思考差异化优势，从身边的头部做起
  • 专注：时间、精力和才华有限；越是优秀，越要专注
  • 迭代：持续迭代、精进；不求妙手，肯下笨功夫

联机学习：找到知识源头，提升认知效率

  • 学习前，想明白学什么、怎么学、有什么用和如何兑现
  • 功利学习法：学得更好，却学得更少；目标导向、及时反馈、最近发展区
  • 知识源头：一手研究论文，行业学术期刊，最新数据报告，大牛最新沟通和思考；问渠哪得清如许，为有源头活水来
  • 知识晶体：关联、树状、序列、数据
  • 联机学习：跨行业学习交流；遇到问题，苦思、找书、找人，提出高质量的问题
  • 终身提问：提问抗折旧，提问即思考，提问即创造；相关性，好奇心
  • 思考角度：证据、视角、联系、猜想、相关
  • 知识IPO：输入问题、解决问题、输出产品；首先提真实、高价值、能被解决的问题
  • 自下而上：站在知识源头，萃取知识晶体；联机学习，交换晶体；以问题为中心学习，创造自己的问题树；用知识IPO让知识变成价值

破局思维：升级思考，解决复杂问题

  • 格局、眼界、系统要大
  • 升维：打破轮回；看得深看得远
  • 系统：系统由元素、元素之间的关系，以及功能三部分构成
  • 回路：搭建正循环系统，学习自增长技能
  • 层级：上层决定下层；下层无解，像上一层
  • ETA脱困四问：情绪-事件-目标-行动
  • 控制点：多层布点，单点可控

内在修炼：跃迁者的心法

  • 真正的改变都是逆人性的
  • 面对世界：开放而专注；求知若饥，虚心若愚
  • 面对自己：迟钝而有趣；多元、超然、好奇心
  • 面对他人：简单、善良、可激怒
  • TFT策略：第一步，合作；以后每一步，重复对手的行动

开放而专注九律

  • 见得人好，经常随喜赞叹
  • 找到自己领域的知识源头，并分享
  • 不随便崇拜谁。一旦崇拜，以他为顶，你的系统就又封闭了
  • 不再认为自己不喜欢、看不懂的东西就是傻的
  • 小心那些“一切能解释得通”的上帝视角感理论
  • 对水平没你高的人要宽容，因为你也没有掌握真理
  • 留出10%-30%的时间，给自己不懂也不太会接触的领域
  • 站在更高角度，发现和专注于自己的独特之处
  • 专注于自己的人生大问题
